# Contributing to `awesome-ai-drug-repurposing`
_Please note that this project is released with a [Contributor Code of Conduct](CODE_OF_CONDUCT.md). By participating in this project you agree to abide by its terms._

We are excited to welcome all contributions from various backgrounds, regardless if you are a senior ML engineer, academic research scientist or an enthusiastic developer. This repository is intended to be a living resource for AI-based drug repurposing work

## Adding something to an awesome list

When contributing, please make sure that:
* the work/publication you are adding is truly awesome _i.e._ you know it well, it's of good quality, and you would personally recommend it to a colleague or friend.
* your work/publication is related to the field of computational/AI-based [drug repurposing](https://www.nature.com/articles/nrd.2018.168).
* you searched through the existing list and you are not duplicating an entry.
* you add the link to the paper (which is not behind a pay-wall).
* you add the link to code/data if it's publicly available.
* you add year of the publication and venue where it was published (if applicable).
* you credit the authors of the work.
* you choose correct section to add the work to.
* Add your name to the list of Authors if you would like!

## Updating your Pull Request

Sometimes, a maintainer of an awesome list will ask you to edit your Pull Request before it is included. This is normally due to spelling errors or because your PR didn't match the awesome-* list guidelines.

[Here](https://github.com/RichardLitt/knowledge/blob/master/github/amending-a-commit-guide.md) is a write up on how to change a Pull Request and the different ways you can do that.
